# Saboteur Demo - Mobile/Friends P0 Task List

> 目标：本项目主要用于「自己和朋友玩」，优先提升移动端体验、对局稳定性与沟通效率。
> 说明：本文件作为后续开发执行清单（给 Codex/开发者对齐范围、优先级、验收标准）。

---

## 背景与范围

- 当前项目已完成核心流程：登录 → 大厅 → 建房/加入 → 开局 → 对局。
- 当前问题集中在移动端：学习成本、误操作、信息过载、弱网语音体验。
- 仅做 **P0（高优先级）**，不做复杂运营系统。

关联现状（代码参考）：
- 登录页当前为输入昵称直接进入大厅：`frontend/src/pages/AuthPage.jsx`
- 局内页已有 `mobilePanel` / `mobileDrawerOpen`：`frontend/src/pages/GamePage.jsx`
- 手牌已支持选中态：`frontend/src/pages/GamePage.jsx` + `frontend/src/components/HandCards.jsx`
- 旋转目前为“双击旋转”：`frontend/src/components/HandCards.jsx`
- 语音为 WebRTC：`frontend/src/contexts/SocketContext.jsx`

## P0 任务清单

## 1) 新手前 3 局引导（可跳过）

### 需求
- 新用户前 3 局进入游戏时，展示 3 步教学：
  1. 拖牌到棋盘
  2. 旋转牌
  3. 弃牌跳过
- 允许“跳过引导”。
- 第 4 局起默认不再弹出（可在设置里手动再次查看）。

### 设计建议
- 使用轻量遮罩 + 聚焦高亮（不影响主流程）。
- 仅在首次进入局内时触发，不阻断 socket 同步。

### 数据与状态
- 本地存储字段建议：
  - `tutorialCompletedCount`
  - `tutorialSkipped`

### 验收标准
- 新账号/新设备首局可看到引导。
- 完成/跳过后进入正常操作。
- 前 3 局可重复提示，第 4 局默认不提示。

---

## 2) 移动端操作容错：显式旋转/弃牌快捷操作

### 需求
- 在移动端，选中卡牌后出现快捷按钮：
  - `旋转`
  - `弃牌`
- 保留双击旋转作为兼容方式，但主引导改为按钮。

### 设计建议
- 快捷按钮跟随选中卡显示，避免遮挡。
- 按钮尺寸满足触控规范（≥ 44px 可点击区域）。
- 旋转后提供微反馈（卡牌轻微缩放/闪光）。

### 验收标准
- 移动端不需要双击即可完成旋转。
- 误触率降低（手动验证：10 次操作无明显误触）。
- 弃牌可通过快捷按钮直接触发。

---

## 3) 局内信息层级压缩（战斗 HUD 模式）

### 需求
- 手机默认仅保留：
  - 当前回合玩家
  - 我的可执行动作（可出牌 / 可弃牌 / 等待）
- 聊天、日志、语音状态、更多玩家信息折叠进二级面板。

### 设计建议
- 复用现有 `mobilePanel` / `mobileDrawerOpen`，形成“主 HUD + 抽屉”。
- 主 HUD 固定区域不超过屏高 25%。
- 日志改为「最近 3 条 + 查看全部」。

### 验收标准
- 主画面第一屏可清楚看到棋盘与当前操作。
- 关键操作（出牌/弃牌）不被聊天和日志挤压。
- 二级面板可在 1 步内打开和关闭。

---

## 4) 语音弱网降级策略（语音失败 → 快捷文本）

### 需求
- 当 WebRTC 连接失败、设备权限拒绝或网络差时，自动降级：
  - 显示提示：`语音不可用，已切换快捷消息`。
  - 提供快捷文本按钮（例如：
    - “我这回合先探路”
    - “我被堵了，求修理”
    - “注意这个人可能是破坏者”
    - “我有地图牌”
  ）

### 设计建议
- 弱网判定触发源：
  - `voiceError` 非空
  - 连续若干次连接失败
- 降级不影响游戏流程，不强制重试。

### 验收标准
- 语音失败时 2 秒内出现降级提示。
- 快捷文本可一键发送到聊天区。
- 无语音权限时仍可顺畅沟通。

---

## 5) 关键行为反馈强化

### 5.1 出牌成功反馈
- 卡牌飞行动画 + 音效 + 轻微震动（移动端）。

**验收标准**
- 出牌后 300ms 内能感知成功反馈。
- 关闭音效/震动后不触发相关反馈（尊重设置）。

### 5.2 关键事件全局条
- 事件示例：堵路成功、修理成功、找到金块。
- UI：顶部全局事件条 + 图标 + 简短字幕。

**验收标准**
- 所有玩家都能看到统一事件提示。
- 提示自动消失，不遮挡主要操作。

### 5.3 轮到我行动增强
- 在现有“轮到你行动了”提示基础上，增加剩余行动时间视觉引导（进度条/倒计时圈）。

**验收标准**
- 当前玩家能明显感知剩余时间。
- 倒计时结束行为与服务器规则一致（若当前无超时规则，则仅作视觉提示）。

## 6) 社交强化（朋友局重点）

## 6.1 房间内快捷表情

### 需求
- 快捷表情建议：
  - “怀疑你是破坏者 👀”
  - “干得漂亮 👍”
  - “救我一下 🙏”
  - “这条路可疑 🤨”

### 验收标准
- 1 步发送，2 秒内所有玩家可见。
- 不需要打开完整输入框即可发送。

## 6.2 一键再来一局（保留同房间）

### 需求
- 结算后增加 `再来一局`，保留当前房间成员与房间号。

### 验收标准
- 房主点击后可直接开启下一局。
- 非房主看到等待状态，避免重复创建房间。
